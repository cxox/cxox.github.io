Git使用

登陆

使用ssh的方式更方便
首先找到用户目录下C:/user/username/.ssh
没有就自己建
然后打开gitbash

ssh-keygen -t rsa [-C 邮箱] // 前面是生成密钥,后面的参数是加密(使用的加密算法),可以不写

git命令

第一次使用

设置全局的姓名和邮箱,可以区分是谁干了什么

git config --global user.name "姓名"
git config --global user.email *********@**.com

常用命令

  • 查看状态

    git status

  • 查看提交记录

    1. 无参数版 
      git log
    2. 图形化显示分支情况 
      git log --graph

  • 来 从网上其它仓库直接克隆

    git clone https:// *** *** ***.git

  • 去

    1. 初始化 
      git init
    2. 都提交到暂存区 
      git add .
    3. 提交到仓库
      1. 提交
      git commit -m "提交描述"
      1. 更正最近一次提交,不产生新的快照
      git commit --amend [-m "新提交描述"]

  • 回

    1. 放弃更改 
      git restore -- <filename> 
//将文件返回提交的模样,也就是放弃修改(这个做不好的话新写的文件就没有了!)
    2. reset
      1. 从暂存区返回 
        git reset HEAD <filename>
// 反悔了,将最近提交的文件从暂存区取回,加上文件名表示只移除特定文件
      2. 从仓库返回:文件状态恢复HEAD后有几个"~“就表示退到多少次提交之前,也可以用数字 HEAD~3 相当于 HEAD~~~
        回到指定的快照直接输入几位识别码就行,例如: 
        git reset 3j2ak43d3f
        1. soft 
          git reset --soft fc234dx
// 只改变仓库内容,工作区和暂存区不受影响
        2. mixed 
          git reset --mixed HEAD~ 
// (默认) 将快照回滚到暂存区,工作区不受影响
        3. hard 
          git reset --hard 243c4dx2werc234
// 将快照还原到当前工作目录,所有区都受影响
      3. 只恢复单个文件 
        git reset 3412qw43 filepath/filename

  • diff

    1. 比较暂存区与工作区(默认) 
      git diff
    2. 比较工作区与仓库 
      git diff 快照id
// 如果是比较最新提交的快照可以直接用 HEAD
    3. 比较两个历史版本 
      git diff 快照id1 快照id2

  • 分支

    • 创建分支 
      git branch [分支名]
    • 切换分支 
      git checkout [分支名]
    • 合并分支,合并到本分支 
      git merge [分支名]
    • 删除分支 
      git branch -d [分支名]
    • 匿名分支:随便使用,不会对文件产生任何影响,可以做任何实验性的事情 
      git checkout HEAD~

  • 与远程仓库的交互

    1. 查看连接状态 
      git remote -v
    2. 建立连接 
      git remote [新连接的名字] [连接的地址]
    3. 连接目标重命名 
      git remote rename [oldName] [newName]
    4. 推送到远程仓库 
      git push [仓库名,例如origin]
